Never ignore them.

---

# Objective

Design and implement the public landing page of Caro.

The landing page is the user's first impression.

It must communicate what Caro is, why someone should use it and encourage visitors to create an account.

The visual language MUST match the authenticated interface already defined in Figma.

The user should immediately recognize that both belong to the same application.

---

# General Rules

Do NOT redesign Caro.

Respect the visual identity already established.

Use the authenticated interface as inspiration.

Everything should feel like one cohesive product.

---

# Hero Section

Create a premium hero section.

Include:

• Official logo (src/logo.png)

• Navigation

• Headline

• Subtitle

• Primary CTA

• Secondary CTA

• Large application preview

The application preview should use the Dashboard interface.

Do NOT use generic illustrations.

The product itself is the hero.

---

# Navigation

Create a sticky navigation bar.

Include:

Logo

Features

About

FAQ

Login

Register

Random Game

The navigation should become slightly blurred while scrolling.

---

# Hero Copy

Do not use generic marketing text.

Create persuasive but clean copy.

Focus on:

Organization

Tracking

Discovery

Progress

Social

Avoid exaggerated claims.

---

# Features

Create a section explaining the application's core features.

Include cards for:

Track Games

Manage Library

Search Thousands of Games

Track Progress

Public Profiles

Friends

Responsive Experience

Each card should include:

Icon

Title

Short Description

Hover Animation

---

# Showcase

Create a showcase section.

Instead of mockups, use actual application previews.

Display previews of:

Dashboard

Library

Search

Profile

Alternate image alignment to create visual rhythm.

---

# How It Works

Create a four-step timeline.

Example:

Search

↓

Add

↓

Track

↓

Share

Each step should contain:

Icon

Title

Description

---

# Why Choose Caro

Create a comparison section.

Show why using Caro is better than manually tracking games.

Do not criticize competitors.

Focus on convenience.

---

# Statistics

Create a section containing visually attractive statistics.

Examples:

Games Tracked

Platforms Supported

Profiles Created

Hours Logged

These values may be placeholders.

---

# FAQ

Create a Frequently Asked Questions section.

Include common questions such as:

Is Caro free?

Can I track console games?

Does Caro support mobile devices?

Can I make my profile public?

---

# Final CTA

Create one final section encouraging users to register.

Keep it simple.

Strong headline.

Short description.

Primary button.

---

# Footer

Every professional website needs a footer.

Include:

Logo

Navigation

Resources

GitHub

Privacy

Terms

Contact

Copyright

# Responsive Design

Desktop

Laptop

Tablet

Mobile

Large Displays

---

# Animations

Use tasteful animations.

Examples:

Hero fade

Cards

Buttons

Scroll reveal

Navbar blur

Hover

---

# Accessibility

Semantic HTML

Proper heading hierarchy

Accessible buttons

Visible focus states

Good contrast

---

# Performance

Optimize images.

Lazy load previews where appropriate.

Avoid unnecessary rendering.

---

# Restrictions

Do not implement authentication.

Do not implement backend.

Do not create fake APIs.

Do not create placeholder lorem ipsum.

Every text should sound like a real product.

---

# Deliverables

The landing page must feel complete.

It should look like the homepage of a real SaaS.

Users should immediately understand:

What Caro is.

Why it exists.

Why they should use it.

Where to begin.
